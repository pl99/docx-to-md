# docx → md

Конвертация Word (`.docx`) в Markdown (`.md`) на чистом `python-docx`.
Заменили `pypandoc`, чтобы получить полный контроль над экспортом таблиц.

## Установка

```bash
pip install -r requirements.txt
```

Зависимости: `python-docx>=1.1.2`, `pytest>=8.0` (только для тестов).

## Использование

### Командная строка

```bash
python docx_to_md.py input.docx output.md
```

Аргументы: ровно два — входной `.docx` и выходной `.md`.
Сообщения об ошибках и успехе выводятся на русском языке.

### Как Python-модуль

```python
from pathlib import Path
from docx_to_md import convert_docx_to_markdown

convert_docx_to_markdown(
    Path("input.docx"),
    Path("output.md"),
)
```

Функция сама создаёт родительскую директорию для `output.md`, если её нет.
Бросает `FileNotFoundError`, если входной файл не существует, и `ValueError`,
если расширение не `.docx`.

## Поддерживаемые элементы

| Элемент           | На выходе                        |
|-------------------|----------------------------------|
| Заголовки H1–H6   | `# ` … `###### `                 |
| Маркированный список | `- `                           |
| Нумерованный список | `1. ` (числовая разметка GFM) |
| Жирный текст      | `**текст**`                      |
| Курсив            | `_текст_`                        |
| Обычный абзац     | как есть                         |
| Простая таблица   | GFM pipe-table                   |
| Сложная таблица   | HTML `<table>`                   |

Жирный и курсив корректно работают внутри одного абзаца и комбинируются.

## Таблицы

### Простые (pipe-table)

Если ни в одной ячейке нет объединения строк или столбцов, таблица
экспортируется в GFM pipe-table.

```markdown
| Name  | Age |
| ----- | --- |
| Alice | 30  |
| Bob   | 25  |
```

Первая строка всегда считается заголовком.
Пустые ячейки выводятся как пробелы между разделителями.
Колонки выравниваются по ширине для читаемости — минимальная ширина
разделителя 3 символа (`---`).

### Сложные (HTML `<table>`)

Если хотя бы одна ячейка содержит `colspan` или `rowspan`, вся таблица
экспортируется в HTML:

```html
<table>
<tr>
  <td colspan="2">Объединённая ячейка</td>
  <td>Обычная</td>
</tr>
<tr>
  <td>A</td>
  <td>B</td>
  <td>C</td>
</tr>
</table>
```

- Все ячейки — `<td>` (не `<th>`).
- Спецсимволы (`<`, `>`, `&`) экранируются.
- Абзацы внутри ячейки склеиваются через пробел.

### Как определяется сложность

Функция `_is_simple_table()` проверяет XML каждого элемента таблицы:

- `w:gridSpan` → есть `colspan` → таблица сложная
- `w:vMerge` → есть `rowspan` → таблица сложная

В противном случае таблица считается простой.

## Тесты

```bash
pytest -v
```

Все тесты создают `.docx` в памяти через `Document()` — файлы на диске
не требуются. Один файл `tests/test_docx_to_md.py` содержит все тесты:
абзацы, заголовки, списки, жирный/курсив, pipe-таблицы, HTML-таблицы,
смешанный контент, CLI-интеграция.

Чтобы запустить конкретный тест:

```bash
pytest -v -k "test_heading"
```
